My process step by step

27/08/2017

Why explaining it with words when we can just draw it?

You decided to create your first website? Here are some tips to begin. You will have an overlook on my process. There are six different steps to go through.

Step 1: Component identification

Depending on the project I am doing, I can start from:

  • a static Photoshop template
  • an Adobe Experience Design template
  • a drawn wireframe
  • scratch

For the first case, I identify directly on the PSD the different components, such as the header, the footer, the articles... I rename the layers in order to organise it as if it was my future code.

In the other cases, I always identify the blocks of code. This is an important part of the process, as it help me to build the organisation of my code. When I know I will build the website with the framework React, this step is needed to see the future components and even states.

Step 2: Mobile First

Nowadays, everyone uses a mobile to check the news, or going on a website. That is why thinking the web-app with the mobile requirements is needed to be user friendly.

After identifying the code blocks that I will write, I draw on a paper how I want my app to appear on a mobile. This will help me to reorganise my blocks and think about what features do I need to add. Indeed there are some components, which doesn't appear on a larger size like the sandwich button for the menu on top of the mobile. This needs to be included in the code way before.

If I know that for the styling I will use a framework like Bootstrap, I will organise my code blocks in the right order and cut it in rows & columns.

Step 3: Time to code!

First of all, I add the project to a source control, which is Git. This is very useful especially when you do a mistake and you want to come back to a previous code. Then accordingly to the plans I have done, I implement it. My code environment is either Sublime Text or Webstorm.

  • HTML5 implementation

    I use the correct HTML5 tags, which correspond to my plans. If I am using the Bootstrap framework, I build my code with rows and columns. I check my work on different browser and debug it if necessary.

  • React implementation

    First of all I import a React boilerplate and set-up my environment. The boilerplate that I use is the one created by Facebook. During my studies, I worked with another boilerplate. The two boilerplates are efficient, but I prefer the one from Facebook as this is the quickest between them.

    Then I build my different components in separate Javascript files. After that I implement my code blocks with and in ES6.

    Finally, I import and export my components.

Step 4: Style your page

For the style I will use CSS3. I create two different style pages: one for the general style and one for the media queries. Everything common to all devices screens such as the colors of the components, the backgrounds or the borders, I will define it parts by parts in the general style sheet. Then I will define my media queries according to four width breaking points:

  • under 767px - for small smartphones screens
  • from 768 to 991px - for bigger smartphones screens, or landscape oriented small smartphones
  • from 992 to 1199px - for tablet screens
  • from 1200px - for desktops

Then everything linked to the size of the components or size is written in the media queries. I build part by part the general style first and then I will define the style rules from the smallest to the biggest media query. At the same time I will try my code in different browsers (Google Chrome, Opera, Mozilla, Safari...) and in the different formats and adjust it.

Step 5: Add the interactions

Now that there are the content and the style of the page, it is time to add some interactions.

I will use either CSS3 or Javascript/jQuery. If the interaction is on one element I implement it with CSS3, for example for the hover effects. I implement more complex interactions with Javascript or jQuery, such as for the carousel effect, the menu appearance in small formats or for modal creation.

The interactions can sometimes not being supported by browsers so this is important to test and debug the code in the browsers.

Step 6: Push the code on production

When everything is working in all formats in all browsers, I will review my code with my mentor in order to improve it. Then I will push the code in production using Git. Finally I will check that everything is alright on different devices: my desktop, my tablet and my phone.

If there are some problems I fix it and make notes on the usability of the page, make some small changes in order to improve the user experience.
